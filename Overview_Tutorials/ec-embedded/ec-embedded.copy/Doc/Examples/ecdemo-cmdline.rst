The parameters are as follows:

.. option:: -f <ENI-FileName>

    Path to ENI file

.. ifconfig:: 'INCLUDE_ECMASTERDEMOMOTION' in ecdemo_cmdline

   .. option:: -cfg <DemoConfigFileName>

      Path to demo configuration file.

.. option:: -t <time>

    Running duration in msec. When the time expires the demo application exits completely.

    :<time>: Time in msec, 0 = forever (default = 120000)

.. option:: -b <cycle time>

    Specifies the bus cycle time. Defaults to 1000 µs (1 ms).

    :<cycle time>:  Bus cycle time in µsec

.. option:: -a <affinity>

    The CPU affinity specifies which CPU the demo application ought to use.

    :<affinity>:    0 = first CPU, 1 = second, ...

.. option:: -v <level>

    The verbosity level specifies how much console output messages will be generated by the demo application. A high verbosity level leads to more messages.

    :<level>:       Verbosity level: 0=off (default), 1..n=more messages

.. option:: -perf [<level>]

    Enable max. and average time measurement in µs for all EtherCAT jobs (e.g. ProcessAllRxFrames).

    :<level>: Depending on level the performance histogram can be activated as well.

.. option:: -log <prefix> [<msg cnt>]

    Use given file name prefix for log files.

    :<prefix>:
    :<msg cnt>: Messages count for log buffer allocation

.. option:: -lic <key>

    Set License key.

    :<key>: License key string

.. option:: -oem <key>

    Use OEM key

    :<key>: 64 bit OEM key.

.. ifconfig:: ('INCLUDE_ECMASTERDEMO' in ecdemo_cmdline or 'INCLUDE_ECMASTERDEMODC' in ecdemo_cmdline or 'INCLUDE_ECMASTERDEMOMASTERRED' in ecdemo_cmdline or 'INCLUDE_ECMASTERDEMORASSERVER' in ecdemo_cmdline or 'INCLUDE_ECMASTERDEMOSYNCSM' in ecdemo_cmdline or 'INCLUDE_ECSIMULATORSILDEMO' in ecdemo_cmdline or 'INCLUDE_ECWINDEMO' in ecdemo_cmdline)

    .. option:: -junctionred

        Enable junction redundancy (automatic mode)

.. ifconfig:: 'INCLUDE_ECMASTERDEMODAQ' in ecdemo_cmdline

    .. option:: -maxbusslaves <cnt>

        Set max number of slaves


.. ifconfig:: 'INCLUDE_ECMONITORDEMO' not in ecdemo_cmdline

    .. option:: -flash <address>

        Flash outputs

        :<address>: 0=all, >0 = slave station address

.. option:: -sp [<port>]

    If platform has support for IP Sockets, this command-line option enables the Remote API Server to be started. The Remote API Server is going to listen on TCP Port 6000 (or port parameter if given) and is available for connecting Remote API Clients.

    :<port>: RAS server port

.. ifconfig:: 'INCLUDE_ECMONITORDEMO' not in ecdemo_cmdline

    .. option:: -rec [<prefix> [<frame cnt>]]

        Packet capture file recording

        :<prefix>: File name prefix
        :<frame cnt>: Frame count for log buffer allocation

.. ifconfig:: 'INCLUDE_ECMONITORDEMO' in ecdemo_cmdline

    .. option:: -rec [<prefix>]

        Packet capture file recording

        :<prefix>: File name prefix

    .. option:: -play <FileName>

        Packet capture file processing

        :<FileName>: File name (\*.pcap|\*.pcapng)

    .. option:: -genebi <FileName>

        Export ENI Builder config on demo stop

        :<FileName>: File name (\*.xml)

    .. option:: -mqtt <BrokerAddress>

        Publish process data to MQTT broker. EcMonitorDemoMqtt only

        :<BrokerAddress>: MQTT broker address

    .. option:: -mqttCycTime <Msec>

        Minimum time between cyclic messages for a variable. EcMonitorDemoMqtt only

        :<Msec>: Time in milliseconds

    .. option:: -mqttChgTime <Msec>

        Minimum time between change-detection based messages for a variable. EcMonitorDemoMqtt only

        :<Msec>: Time in milliseconds

.. ifconfig:: 'INCLUDE_ECMONITORDEMO' in ecdemo_cmdline or 'INCLUDE_ECMASTERDEMODAQ' in ecdemo_cmdline

    .. option:: -daqrec <FileName>

        :<FileName>:  Configuration file

.. ifconfig:: 'INCLUDE_ECMASTERDEMODC' in ecdemo_cmdline or 'INCLUDE_ECMASTERDEMOMOTION' in ecdemo_cmdline

    .. option:: -dcmmode <mode> [<synctocyclestart>]

        Set DCM mode

        :<mode>: off | busshift | mastershift | masterrefclock | linklayerrefclock
        :<synctocyclestart>: Sync to cycle start: 0 = disabled (default), 1 = enabled
    
    .. option:: -dcmlog
    
        Enable DCM logging (default: disabled)
        
    .. option:: -ctloff

        Disable DCM control loop for diagnosis
